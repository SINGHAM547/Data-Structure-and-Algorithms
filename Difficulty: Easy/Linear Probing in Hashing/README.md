<h2><a href="https://www.geeksforgeeks.org/problems/linear-probing-in-hashing-1587115620/1">Linear Probing in Hashing</a></h2><h3>Difficulty Level : Difficulty: Easy</h3><hr><div class="problems_problem_content__Xm_eO" style="--text-color: var(--problem-text-color);"><p data-start="259" data-end="577"><span style="font-size: 14pt;">Given an array <strong>arr[] </strong>of integers and a hash table of size <strong>m,</strong> insert the elements of the array into the hash table using <strong>linear probing</strong> to handle collisions. Duplicate elements must be mapped to the same position in the hash table, while colliding elements must be mapped to the position <strong>[(value+1) % m]</strong>.&nbsp;</span></p>
<p data-start="579" data-end="668"><span style="font-size: 14pt;"><strong data-start="579" data-end="588">Note:</strong> If there is no space left to insert a new element, simply <strong>discard</strong> that element.</span><br><span style="font-size: 14pt;"><strong>Linear probing</strong> is a collision-handling technique used in hashing. According to linear probing, whenever a collision occurs, the algorithm searches for the next immediate available position.</span></p>
<p><span style="font-size: 14pt;"><strong>Examples:</strong></span></p>
<pre><span style="font-size: 14pt;"><strong>Input:</strong> arr[] = [3, 11, 10], m = 3
<strong>Output: </strong>[3, 10, 11]<strong>
Explanation: <br></strong>3 % 3 = 0, So insert 3 in hashtable[0],<br>11 % 3 = 2, So insert 11 in hashtable[2], <br>10 % 3 = 1, So insert 10 in hashtable[1].  </span></pre>
<pre><span style="font-size: 14pt;"><strong>Input: </strong>arr[] = [4, 14, 24, 44], m = 10
<strong>Output: </strong>[-1, -1, -1, -1, 4, 14, 24, 44, -1, -1]<strong>
Explanation: <br></strong>4 % 10 = 4, So insert 4 in hashtable[4],<strong><br></strong>14 % 10 = 4, but hashtable[4] is already filled Since there is a collision, use a linear Probing to find the next available slot. <br>Insert 14 into next available position which is hashtable[5]. <br>Continue the process for remaining element.</span></pre>
<pre><span style="font-size: 14pt;"><strong>Input: </strong>arr[] = [5, 5, 5, 5, 5], m = 5 <strong><br>Output: </strong>[5, -1, -1, -1, -1]<strong><br>Explanation: <br></strong>5 % 5 = 0, So insert 5 in hashtable[0],<br>Next, 5 % 5 = 0, but hashtable[0] is already filled and since number is same, it is mapped to the same position in the hash table.</span></pre>
<p><span style="font-size: 14pt;"><strong>Constraints:</strong><br>1 ≤ arr.size() ≤ 10<sup>4<br></sup></span><span style="font-size: 14pt;">1 ≤ arr[i] ≤ 10<sup>5<br></sup></span><span style="font-size: 14pt;">1 ≤ m ≤&nbsp;1000<sup><br></sup></span></p>
