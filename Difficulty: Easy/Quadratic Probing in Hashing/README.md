<h2><a href="https://www.geeksforgeeks.org/problems/quadratic-probing-in-hashing-1587115621--173701/1">Quadratic Probing in Hashing</a></h2><h3>Difficulty Level : Difficulty: Easy</h3><hr><div class="problems_problem_content__Xm_eO" style="--text-color: var(--problem-text-color);"><p><span style="font-size: 18px;">Given an array<strong> arr[] </strong>of integers and a hash table of size <strong>m,</strong> insert each element of the array into the hash table using <strong>Quadratic Probing</strong> for collision handling.</span></p>
<p><span style="font-size: 14pt;"><strong>Note: </strong></span><span style="font-size: 18.6667px;">All the positions that are unoccupied are denoted by <strong>-1</strong> in the hash table.<br></span><span style="font-size: 14pt;"><strong data-start="135" data-end="156">Quadratic Probing</strong> is a collision handling technique used in hashing. When a collision occurs (i.e. two elements hash to the same index), Quadratic Probing searches for the next available slot using a quadratic function of the form: </span><strong style="font-size: 18.6667px;">newIndex = (originalIndex + i<sup>2</sup>) % m&nbsp;</strong><span style="font-size: 14pt;">(i = number of collisions seen so far for this insertion)</span></p>
<p><span style="font-size: 18px;"><strong>Examples : <br></strong></span></p>
<pre><span style="font-size: 18.6667px;"><strong>Input: </strong>arr[] = [21, 10, 33, 43], m = 5<strong><br>Output: </strong>[10, 21, -1, 33, 43]<strong><br>Explanation:<br></strong>21 % 5 = 1, So 21 goes into hashtable[1],<br>10 % 5 = 0, So 10 goes into hashtable[0],<br>33 % 5 = 3, So 33 goes into hashtable[3],<br>43 % 5 = 3, but hashtable[3] is already filled so we try (3 + 1<sup>2</sup>) % 5 = 4, So 43 goes into hashtable[4] because it is empty.</span></pre>
<pre><span style="font-size: 14pt;"><strong>Input:</strong> arr[] = [31, 48, 37, 70], m = 4<br><strong>Output:</strong> [48, 37, 70, 31]<br><strong>Explanation:<br></strong>31 % 4 = 3, So 31 goes into hashtable[3],<br>48 % 4 = 0, So 48 goes into hashtable[0],<br>37 % 4 = 1, So 37 goes into hashtable[1],<br>70 % 4 = 2, So 70 goes into hashtable[2].</span></pre>
<p><span style="font-size: 18px;"><strong>Constraints:</strong></span><span style="font-size: 18px;"><sup><br></sup>2 ≤ m ≤ 10<sup>4</sup><br>1 ≤ arr.size() ≤ m<br></span><span style="font-size: 18px;">1 ≤ arr[i] ≤ 10<sup>6</sup><sup><br></sup></span></p>
